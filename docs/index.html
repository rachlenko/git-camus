git-camus

Transform your Git commit messages into philosophical reflections inspired by the works of Albert Camus.

Welcome to Git-Camus, the utility that transforms your Git commit messages into philosophical reflections inspired by the works of Albert Camus. Why settle for mundane commit messages when you can make each one a profound meditation on absurdity, rebellion, and the human condition?

With Git-Camus, the simple act of version control becomes an opportunity to reflect on life's paradoxes, turning every push, pull, and merge into a philosophical journey.

Here's how it works:

git_camus -m "Initial commit"

Instead of a bland commit message, you'll get something like:

"The code compiles, but does it solve the eternal question of why we code at all?"

Whether you're exploring the absurdity of debugging or the rebellion against technical debt, Git-Camus brings meaning to your workflow. Now, each commit is not just a record of changes, but a mark of your existential exploration.

Quick Start

Install git-camus

pip install git-camus

Configure Model and Prompt

Create a config.toml file in your project or home directory:

[run]
model_name = "llama3:70b"
prompt_message = "You are an AI assistant... (customize your prompt here)"

Run Locally

git-camus

Run with Docker

# Build the Docker image
 docker build -t git-camus .

# Run with default settings
 docker run --rm -v $(pwd):/app git-camus

# Run with a custom config file
 docker run --rm -v $(pwd):/app -v $(pwd)/config.toml:/app/config.toml git-camus

# Override the model via environment variable
 docker run --rm -v $(pwd):/app -e OLLAMA_MODEL=llama3:8b git-camus
